[ht::ug::top /tools/ColorTableEditor.html] [ht::overview {The color table editor allows you to view, modify and create color tables.} ]
  1. Open up the Color Table Editor from the [ht::menu Tools {Color Table Editor}] menu.
  2. You should see something like this (well, plus a Brightness field):
    3. [ht::cimg images/Overview.jpeg -popup 1] The Color Table Editor consists of:
    • A set of command menus: [ht::menu File], [ht::menu {Color Tables}] and [ht::menu Help].
    • A color bar which shows the currently selected color table.
    • A set of [ht::index breakpoints] that allow you to manipulate the color bar. Breakpoints are a color table editing convenience. You can add breakpoints to a color table, move them and remove them. You change the color in the color table directly by moving a breakpoint or by filling and/or interpolating between two breakpoints.
    • A color chooser which allows you to select a color.
    • A set of edit mode items.
    • History buttons that allow you to undo and redo changes.
  3. First, we want to select a color table.
    • Select the [ht::menu {Color Tables}] menu:
      • [ht::cimg images/Bright38Menu.gif]
    • Select [ht::menu Basic Bright38].
  4. We don't want to change this particular color table so let's make a copy of it by saving the currently viewed color table under a new name.
    • Select the [ht::menu File {Save As...}] menu item.
    • Enter the name for the new color table: [ht::qt test1].
  5. Now we'll play around a little changing the colors through direct manipulation of the breakpoints.
    • Click the [ht::button Fill] edit mode button. You can select a color to fill with from the Color Chooser at the bottom of the window.
    • Click on the breakpoint on the left and drag it around.
    • Try changing colors in the color chooser.
    • Click the [ht::button Interpolate] edit mode button and drag a breakpoint around.
    • Click the [ht::button Brightness] edit mode button and change the brightness to 50%. Drag a breakpoint around.
    • Click the [ht::button Transparency] edit mode button, select 50% transparency from [ht::button Transparency] menu and drag a breakpoint around.
    • Click on the [ht::button History] back and forward arrows to undo/redo any changes you make.
    • Click the [ht::button +] and [ht::button -] signs below the color bar to change the number of color slots in the color table.
  6. Let's start fresh.
    • Select the [ht::menu File Remove] menu item. This deletes the currently viewed color table.
    • You should be viewing the initial color table.
    • Select the [ht::menu File {Save As...}] menu item and enter the name for a new color table: [ht::qt test1].
    • Set the edit mode to [ht::command None]
    • Now we'll change the colors through the [ht::menu Breakpoint] menu.
      • Add a new breakpoint.
        • Right click in the middle of the color bar to bring up the [ht::menu Breakpoint] menu:
          • [ht::cimg images/AddBreakpointMenu.gif]
        • Select [ht::menu {Add Breakpoint} Here].
        • You should have a color bar that looks something like:
          • [ht::cimg images/NewBreakpoint.gif]
      • Right click on the new breakpoint and we will get a different menu:
        • [ht::cimg images/SetColorsMenu.gif]
      • Select [ht::menu {Interpolate Right}].
      • Select [ht::menu {Fill Left}].
      • etc.
  7. Let's get back to the beginning.
    • Select the [ht::menu File Remove] menu item. This deletes the currently viewed color table.
