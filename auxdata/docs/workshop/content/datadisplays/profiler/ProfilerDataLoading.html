NOAA Profiler Network data can be accessed in the IDV from ADDE servers. You can load and display several stations and times at once.

  1. Setup for Profiler displays
    • [ht::clearDisplaysAndData].
    • Uncheck the [ht::menu Projections {Auto-set Projection}] menu option.
    • Set the projection to Kansas ([ht::menu Projections Set Kansas] menu).
  2. Open the Data Source Chooser for NOAA Profiler Network Data
    • [ht::openDataSourceChooser]
    • In the [ht::dialog Data Source Chooser] click on the [ht::command Profiler] tab to see the pane for access to NOAA National Profiler Network data on ADDE servers.
    • The map shows Profiler station names. Zoom in to see all stations in Oklahoma and Kansas. [ht::mapSelector]
    • Select the four stations Hillsboro, Haviland, Neodesha, and Lamont, one at a time, by holding the Ctrl key down while clicking on the station name. You can put one or more stations in any IDV Profiler display.
      You can de-select stations with Ctrl-click too. You can select a group of stations by a rubber band box. Hold the Shift key down, then left click and drag the mouse. This only selects stations shown on the map; make sure decluttering is turned off to get all stations in a region.
    • In the [ht::command Server] box, use the pull down list to select a remote ADDE server with Profiler data. Select adde.ucar.edu.
    • Select [ht::command Hourly] in the [ht::command Data Interval] box. You can choose from Hourly, 30 minute, 12 minutes, or 6 minute. This sets the time intervals in the display.
    • Click on [ht::command Connect]. A list of times will appear; the most recent 12 times are pre-selected.
    • Select the most recent 6 times available. (Shift-click on the sixth time up from the bottom.)
    • Click on [ht::command Add Source] to connect the IDV to the data for this Profiler data request.

    The label [ht::command Profiler Hourly - 4 stations] appears in the [ht::command Data sources] panel in the [ht::dialog Field Selector], and [ht::param Profiler winds] appears in the [ht::dialog Fields] panel.
