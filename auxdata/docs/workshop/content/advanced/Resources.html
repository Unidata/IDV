So, how do you put all of this together?
  1. Remember all of those resources we created in earlier exercises? They are saved off in the user's local resource directory: [ht::popup file:<%workshop.homedir%>/.unidata/idv/DefaultIdv [ht::filename <%workshop.homedir%>/.unidata/idv/DefaultIdv]].
  2. The files include:
    • [ht::filename aliases.xml] - The data aliases.
    • [ht::filename default.xidv] - The default bundle.
    • [ht::filename main.xml] - User preferences.
    • [ht::filename paramdefaults.xml] - Parameter defaults.
    • [ht::filename projections.xml] - Projections.
  3. The IDV looks in the user's resource directory, the site path and the system path for these resources.
  4. View the available resources
    • From the [ht::menu Tools] menu select Resource Manager.
    • This provides an overview of all of the available individual resources.
    • But where are these individual resources kept? [ht::more "
    • To see just where the resources are located select the [ht::button List Sources] button." "Answer..." ""]
  5. To share these resources across a site one just needs to copy these files to an accessible web site or directory. Then point the IDV to that path with the -sitepath command line argument. Try running the IDV with: 
    idv -userpath ~/otheridv -noplugins -listresources -sitepath https://www.unidata.ucar.edu/software/idv/workshop
  6. Note: when running with the -sitepath argument that value is persistent. Bring up the Preferences and remove the sitepath entry.
  7. Exit the IDV.
