Integrated Data Viewer
Version <%idv.version%>
The items below reflect changes since the 3.0u2 release. Upon IDV start up, "PermGen" memory allocation is done automatically according to the user's machine specifications. The default PermGen memory allocation can be changed through the User Preferences ( [ug::menu Edit Preferences] menu) under the System tab. A new version of the netCDF-Java library (4.2.34) is included in this release. See the netCDF-Java change log for a complete list of changes. In the area of publication quality graphics, there are two new ways to provide latitude and longitude labels on each view. First, you can place labels along the edges of the box axes through the [ug::menu View Properties] menu under the [ug::button {Horizontal Scale}] tab. The second option allows you to place labels directly in the view through the Map Display Control. Also, in the area of publication quality graphics, the IDV has new multi-view image and movie capture. There is now an All Views option available both movie and image capture via View->Capture. These options are also available in the ISL. Various minor bug fixes and improved handling of fonts and overall usability. Fixed broken webcam links available in the IDV. Fixed broken image transparency issues that emerged when we introduced multi-image capture in a previous IDV version. The IDV can now match display times. This is done by first setting a display to be the 'time driver', and setting a second display to 'Use Time Driver Times' or 'Match Time Driver'. A display can be set as the time driver or to use time driver times in the following locations:
  • Data Choosers tab for Satellite Imagery, Radar (Remote and ADDE Level III), and Observations (Point and Profiler)
  • Field Selector tab
  • View menu of the Displays Control tab
 The IDV Scripting Language (ISL) has new capability to export data available in ObsListControl (e.g. a Point Data List display for METAR data). The IDV Scripting Language (ISL) image tag has a new animation_index attribute to specify multiple images in the animation. There is a new major RAMADDA section in IDV training workshop.
