The IDV can display non-geolocated images and geolocate them.
  1. First, start fresh [ht::refresh]
  2. Load an image.
    • [ht::openDataSourceChooser]
    • In the [ht::dialog Data Chooser] tree click on the [ht::command URLs] node.
    • Enter the URL:
      https://www.unidata.ucar.edu/software/idv/data/image.gif
    • Press [ht::button Add Source]
    • The IDV will take some time to load this because it defaults to the VisAD data source.
  3. Let's look at this image.
    • You should have the image loaded in the [ht::dialog Field Selector] Note the funny field name. This is the VisAD math type of the data.
    • Create an [ht::display Omni Control] display to see the image.
    • Click on the [ht::button Mappings] button.
    • Let's play around with some of these mappings.
  4. Now we're going to look at the moon.
    • In the [ht::dialog Data Chooser] tree click on the [ht::command URLs] node and enter the URL:
      https://www.unidata.ucar.edu/software/idv/data/moon.gif
    • Let's geolocate the image.
      • In the [ht::command Data Sources] in the [ht::dialog Field Selector] window select [ht::param Formulas]
      • Under [ht::command Fields] open the [ht::param Imagery] tab. Select [ht::param Geolocate an Image].
      • Under [ht::command Displays] select [ht::display 3 Color (RGB) Image] and create the display.
      • In the popup enter: 90,-180,-90,180 and press [ht::button OK]
      • Now, select the funny [ht::param (ImageElement,ImageLine)...] field.
    • Now, a flat moon picture isn't very exciting. Let's create a Globe display window with the [ht::menu File New {View Window} {Globe Display} {One Pane}] menu item.
    • Create the moon image as described above.
