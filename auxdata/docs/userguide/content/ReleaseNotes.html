Integrated Data Viewer
Version <%idv.version%>
The items below reflect changes since the 2.5 release. The IDV now uses the Java-netCDF 4.0 library. This provides limited support for HDF-4 and HDF-5 files. The IDV installers now use Java 6. Web Start still uses Java 5 (JDK 1.5) since there is no version of Java 6 on Mac OS-X for older systems. The default location for the storage of user data (preferences, favorite bundles, plug-ins, etc) has been changed from .metapps to .unidata/idv under the user's home directory. This is consistent with other Unidata software (e.g. netCDF-Java) which use the .unidata directory. Any display can now be saved in a vector graphics (PDF, PostScript, Scalar Vector Graphics (SVG)) format. Use the [ug::menu View Capture Image] menu and use .pdf, .ps or .svg for the file suffix to create the appropriate format. This works best for overhead, line plots. For 3D plots, raster (e.g. GIF, PNG, JPEG) is still the best option. This work was made possible using code from the Australian Bureau of Meteorology. You can now set the dash pattern for contours below the base contour. Select the dash pattern from the Contour Dialog. The chooser for the Level II data from the THREDDS Data Server (TDS) can now be used to access Level III and Terminal Doppler Weather Radar (TDWR) products as well. This provides faster access than the ADDE server and allows you to access data across days. Point data (observations) can now be gridded using a Barnes objective analysis routine. Numeric fields are listed under a "Gridded Data" category in the Field Selector. Users can control the analysis parameters through the Data Source properties or through the Data Subset panel. This work was made possible using code from the Australian Bureau of Meteorology. Users can now access weather text products from a remote server or their own local archives. The server products use the ADDE protocol. The local archives are based on the GEMPAK NWX tables. For the latter, you must have GEMPAK installed and have the appropriate environment variables defined for the NWX program. Information on using the display control can be found in the Text Product Control documentation. Users can access the Automated Tropical Cyclone Forecasting (ATCF) data from the National Hurricane Center in the IDV. This archive has historical storm tracks back to the 1800's. Use the [ug::menu Data {Load Data} {ATCF Storm Tracks}] menu to access this feature. The resulting display allows you to compare model forecasts in the database. For a list of outstanding known problems, see the Known Problems page.
