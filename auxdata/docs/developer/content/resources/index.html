Macros and Resource Paths

The IDV has a general way of dealing with paths to property files, resource files, etc. First, a path can be one of:
  • A file path (e.g., /home/jeffmc/.metapps/DefaultIdv/colortables.xml)
  • A url (e.g., https://www.unidata.ucar.edu/software/idv/resources/colortables.xml)
  • A Java resource path (e.g., /ucar/unidata/idv/resources/colortables.xml)
It also makes use of a macro facility to take some generic path that contains a macro and, expanding the macro, use the expanded path as the actual resource path. The available macros are:
%USERPATH% The file system path of the user's .metapps/application directory
%SITEPATH% If defined (usually by the -sitepath argument) the directory path or url of where to find site resources
%IDVPATH% The internal (to the jars) /ucar/unidata/idv/resources path
%APPPATH% This is normally the java resource path of the package of the instantiated idv object. For the example idv application it would be: /ucar/unidata/apps/example
So for example, the above example paths may actually be defined as:
  • /home/jeffmc/.metapps/DefaultIdv/colortables.xml
    is actually defined as:
    %USERPATH%/colortables.xml
    where %USERPATH% is:
    /home/jeffmc/.metapps/DefaultIdv
  • https://www.unidata.ucar.edu/software/idv/resources/colortables.xml
    might be defined as:
    %SITEPATH%/colortables.xml
    where %SITEPATH% is:
    https://www.unidata.ucar.edu/software/idv/resources
  • /ucar/unidata/idv/resources/colortables.xml
    is really:
    %IDVPATH%/colortables.xml
    Where: %IDVPATH% is:
    /ucar/unidata/idv/resources
