[ht::desc In this section we will cover how to save the state of the IDV in a [ht::qt bundle] file.]
In this section we will cover how to save the state of the IDV in a bundle file. Bundles are xml files that hold all of the application state of the IDV: data sources, displays, windows and view state.
  1. First, clear all existing data and displays.
  2. Load in a data source and create a display.
    • [ht::loadBasicGrid]
    • [ht::createBasicDisplay -display {Color-Shaded Plan View}]
    • Rotate the display in the main window and reposition the control window and the main window.
  3. Save this state off as the [ht::filename workshop.xidv] bundle file.
    • Select the [ht::menu File {Save As...}] menu from the main menu bar.
    • Under the "What should be saved" leave everything checked.
    • Under the "Jython library" select "None".
    • [ht::save workshop.xidv]
  4. You can use a bundle in a variety of ways:
    • Open a bundle while running.
      • Remove all displays and data. Reposition the main window.
      • Select the [ht::menu File Open] menu item.
      • Select the [ht::filename workshop.xidv] file.
      • You will be prompted whether you want to remove the current data and displays. Press [ht::button OK].
      • You should see the state that was saved in the bundle.
    • You can also provide the bundle file as a command line argument when starting the IDV, e.g.: 
      ./runIDV <%workshop.homedir%>/workshop.xidv
