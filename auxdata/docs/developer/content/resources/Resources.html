\n" } ] The IDV makes use of the [dg::class ucar.unidata.util.ResourceCollection] and [dg::class ucar.unidata.xml.XmlResourceCollection] classes to hold different collections of resources. These resources are defined in the property file. The default IDV resources are kept in the package [dg::package ucar.unidata.idv.resources]. The example properties file has documentation about how to define new paths to resources.

In the properties file is a set of properties:

application.resource.paramgroups=%IDVPATH%/paramgroups.xml;%SITEPATH%/paramgroups.xml;
application.resource.derived=%USERPATH%/derived.xml;%IDVPATH%/derived.xml
made up a of list of semi-color separated paths. These paths can refer to the local file system, to internal java resources or they can be urls.

Many of these resource files can be edited by the end user through the IDV, including colortables, parameter defaults, data aliases, derived data, Jython libraries and users choosers. For these resources the first resource specified in the properties file is the one that is editable. For example, from [dg::class ucar/unidata/idv/resources/idv.properties], is the line:

application.resource.colortables=%USERPATH%/colortables.xml;%IDVPATH%/colortables.xml
The first resource, %USERPATH%/colortables.xml, is where user created and edited color tables get stored. The macro %USERPATH% is replaced with the path to the idv preference directory:
userhomedirectory/.metapps<application class name>
which is where all user created or modified data is stored.

If you want to release an application with some new or altered resource files you can simply bring up the IDV, edit the given resource (e.g., through the Edit menu). Then simply look in your .metapps directory for the relevant file and copy that into your application area (or on a web server).

[resource {Data source specifications} application.resource.datasource datasource.xml] The Data source specification resources define the set of DataSource classes available and what kinds of files and/or urls they can handle. See datasource.html for more information. [resource {Display controls} application.resource.controls controls.xml] This resource defines the different displays available with the IDV and what kinds of data they are applicable to. See displays.html for more information. [resource {Derived data} application.resource.derived derived.xml] These resources are used to define end-user formulas and default derived quantities. This is a pretty complicated piece of the IDV and is described more fully in derived.html [resource {Color tables} application.resource.colortables colortables.xml] The XML that this resource points to is not a custom XML format, rather it is the result of persisting (using the XmlEncoder persistence mechanism) a list of [dg::class ucar.unidata.util.ColorTable] objects that have been created with the color table editor (See the Edit->Color tables menu). [resource {Projections} application.resource.projections projections.xml] The projections.xml file is like the colortables.xml in that it too is the result of XmlEncoder persitence. To change projections or create new ones go to the View menu and choose Projections. Edit the list of projections. A new projections.xml file will be saved in your .metapps directory. [resource {Default bundles} application.resource.bundles default.xidv] This list of resources points to where default bundles (those that are automatically loaded by the IDV) are kept. [resource {Data choosers} application.resource.choosers choosers.xml] The choosers.xml file allows you to define what kinds of data choosers show up in the data chooser window. See for more detail. [resource {User choosers} application.resource.userchooser userchooser.xml] This is probably going to become obsolescent in the future but the userchooser.xml holds the user created data chooser entries (See the "New pane" button on the data chooser window) [resource {Jython libraries} application.resource.jython default.jy] The Jython library is used to initialize the Jython interpreter. [resource {Menu bar} application.resource.menubar defaultmenu.xml] The menu bar used within the IDV is defined via xml. }] For menu items there is a (String) action which gets passed to the [method IntegratedDataViewer.handleAction] method. If the action starts with "jython:" the rest of the string is executed by the Jython interpreter (For security only methods calls of the form: idv.method(...) are allowed).
