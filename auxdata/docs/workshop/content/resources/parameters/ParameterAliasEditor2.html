[ht::desc More on parameter aliases.]
  1. The answer is:
    • The color table is the [ht::ct default] color table.
    • The range used is the range from the actual data.
  2. The problem is that the IDV does not recognize that the name of the [ht::param temperature at fixed height above ground] field as a temperature.
  3. To solve this problem, the IDV provides a data alias facility.
  4. First we need to know the actual parameter name, not its description.
    • Select the [ht::menu Help Details] menu item from the [ht::display Color-Shaded Plan View] display control that was just created.
    • You should see a detailed description of this display, including the data it uses:
      • ...
 T_fhg temperature at fixed height above ground from Geogrid data source
...
    • The parameter name is [ht::param T_fhg].
  5. Open up the [ht::dialog Alias Editor] from the [ht::menu {Tools} {Parameter Alias Editor} ] menu.
    • You should see 2 tabs: [ht::command User Aliases] and [ht::command System Aliases]
    • The first tab shows the user's editable aliases. The second tab shows the default system aliases.
    • Select the [ht::command System Aliases] tab.
    • Right click on the [ht::param TEMP] row and select [ht::menu {Copy to User Table}].
    • You should now see this row in the [ht::command User Aliases] tab and the Data Alias dialog should be shown.
    • Scroll down to the bottom of the [ht::command Aliases] list, enter [ht::qt T_fhg] and click the [ht::button OK] button.
    • Click the [ht::button OK] button in the [ht::dialog Alias Editor] window.
    • To edit an existing entry right click or double click on the parameter.
  6. Let's see if this works.
    • Remove your displays by selecting the [ht::menu Edit {Remove All Displays}] menu item.
    • [ht::createBasicDisplay -display {Color-Shaded Plan View} -param {temperature at fixed height above ground}]
    • You should see the [ht::ct Comfort] color table and the correct range being used now.
