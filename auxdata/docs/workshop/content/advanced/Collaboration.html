The IDV supports real-time, multi-user collaboration and sharing.
  1. Collaboration is based on one or more IDV's being connected together. This topology can be

    peer-to-peer: [ht::cimg peertopeer.gif] multiple-client/single-server: [ht::cimg clientserver.gif] or a heterogeneous structure: [ht::cimg mixed.gif]

  2. To start a collaborative session, open the [ht::dialog Collaboration] window by selecting the [ht::menu Collaboration {Show Window}] menu item.
  3. The [ht::dialog Collaboration] window is composed of four tabs:
    4. [ht::cimg CollabGui1.gif]
    • The [ht::command Connect] tab allows you to connect to other machines or to start up a server on your machine.
    • The [ht::command Clients] lists what other users you are connected with.
    • The [ht::command Chat] tab provides for text chat with other users.
    • The [ht::command Log] tab shows a log of messages.
  4. Remove any displays you may have [ht::refresh] and connect to my machine by entering the presenter's computer's hostname in the [ht::command Connect to] field and press return.
  5. You can see the current connections under the [ht::command Clients] tab.
    6. [ht::cimg CollabGui2.gif]
    • You can turn on/off whether changes you make are sent to others with the [ht::button Ok to send changes] checkbox.
    • You can turn on/off whether you receive other user's changes with the [ht::button Ok to receive changes] checkbox.
    • The [ht::button Ok to relay changes] checkbox is used when your IDV is acting as a server
    • The checkboxes in the [ht::command Clients] list allow you to turn on or off the sending and receipt of changes for individual users.
  6. I am going to load in some data and create some displays.
  7. Now, I am going to disconnect all of you from my machine. I want you to connect up in a peer to peer fashion with your colleagues.
    • Some of you will act as a server. Go to the [ht::command Connect] tab and click the [ht::button Start server] button.
    • Some of you will act as a client. Go to the [ht::command Connect] tab and enter the hostname of the machine you want to connect to.
    • For the server machines you should see a dialog window popping up asking if it is OK to accept the connection. Click the [ht::button Yes] button.
    • Now, play around a bit. Load in some data, create displays, change their state, enter some text in the chat window, etc.
