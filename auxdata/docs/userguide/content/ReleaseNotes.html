Integrated Data Viewer
Version <%idv.version%>
The items below reflect changes since the 2.8u1 release. The IDV installers have been updated to Java 6 Update 20. This version of the IDV is using the final release of version 4.2 of the netCDF-Java library. This new version includes several bug fixes for GRIB and point data reading. Selected grid displays (contours, flow vectors, wind barbs) now have an option to perform smoothing functions on the grid. The GEMPAK 5-point (SM5S), 9-point (SM9S) and Gaussian Weighted Function (GWFS) smoothing functions have been implemented in the IDV. Users can select one of these functions in the display control window and/or save these as a Display Setting to use when creating the display. A new feature to capture time height wind profiler images has been added, users can now capture the wind profiler image in the IDV. A new point cloud display was added to the some point data sources to support displaying a large number of points quickly. New grid diagnostics are available in this release. The following have been added:
  • PVOR(S,V) - Potential Vorticity
  • QVCL(THTA, V) - Q-Vectors in a layer
  • SAVG,SAVS - average over a grid
  • SM5S, SM9S, SM5V, SM9V, GWFS, GWFV - grid smoothers (5-point, 9-point, Gaussian)
You can use the Grid->Define a grid diagnostic formula to use these functions. When GRIB files are read from disk, an index is created to speed up access to the individual records. This was always created in the temporary cache under the DefaultIdv/tmp/nj22 directory. However, that cache gets scoured after a couple of days, the the indexes need to be recreated the next time you load the data. There is now an option the User Preferences under the Formats & Data tab to control where the indexes are written. By unchecking the "Write Grib Index in Disk Cache" option, the indexes will be written to the same directory as the files and will not be scoured. A new profiler data source type was added to the IDV to support reading of local netCDF profiler data including CAP and NPN datasets. A radar data type was added to the IDV to support for the WSI NOWrad images in our archive case studies datasets. Several bug fixes were made in this release, including:
  • Fix Java 3D installation in Mac OS-X installer
  • Remove dependency on Java 3D 1.5.2 that was added in the 2.9u1 release
  • netCDF-Java library not correctly handling scale/offset on 20th Century Reanalysis datasets
  • Mapping problems for global grids
For a list of outstanding known problems, see the Known Problems page.
