Version <%idv.version%>
The items below reflect changes since the 5.2 release.

The version of the netCDF-Java library currently distributed with the IDV is the 4.6.5. This is a bug fix release. See the netCDF-Java Library for a more details information.

The IDV can now display 3D volume vectors based on the x, y, and z wind component fields, the unit of the z componet wind could be in meter per second or in pascal per second.

The 3D volume trajectory is based on the new APIs of the VisAD library enhancements with much better performance. The volume trajectory display is categorized as 3D flow displays that also includes 3D vecotr cross section, 3D volume vector, 3D volume vector colored by speed, and 3D volume vector colored by another parameter. The 3D volume vector display could be created first and there is radio button to switch to the volume trajectory display. The start points of volume trajectory could be the whole volume of the gridded dataset or the selected level of the gridded dataset.

In addition to the traditional line trajecory display, ribbon, cylinder and deform ribbon are added as new trajectory dispaly types and are available in this release.

The local McIDAS area files can be calibrated and spatially subset, currently only for the GVAR (GOES) area files in this release and we are working on area files from other satellites.

We fixed a few bugs associated with the time matching in this release. One of the bug is the excessive loading of data in the bundle when there is a derived of derived parameters. The another bug fixed is in the 3D grid trajectory display in which the time subset information not correctly saved in the bundle.

For a list of outstanding known problems, see the Known Problems page.
